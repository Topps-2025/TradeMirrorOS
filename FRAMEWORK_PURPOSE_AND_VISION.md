# 框架目的与愿景

更新日期：2026-04-11

## 一、这套框架到底要解决什么问题

这不是一个“帮你荐股”的系统，也不是一个纯资讯聚合器。

它首先是一个面向主观 / 半量化交易者的交易行为复盘记账本。

同时，它也开始为“量化 / 半量化但仍带有人为主观开关”的场景预留单独轨道：

- 策略代码可以固化选股和执行
- 但因子怎么选、哪条策略今天启用、参数版本是否切换、是否做人为降杠杆，依然带有主观判断
- 这类信息不应该被硬塞回传统“选股理由一句话”里，而应该形成独立的策略上下文与轨迹

它要解决的核心问题是：

- 盘中先做了，盘后却很难把当时为什么做、为什么慌、为什么偏离计划完整记下来
- 人会遗忘自己的失败样本，尤其容易只记住赚钱时的逻辑
- 很多交易经验停留在聊天、截图和模糊印象里，无法形成可复用资产
- 人工复盘太费劲，很难长期坚持

所以，这个框架的目标不是替你做决策，而是把你的交易轨迹变成可沉淀、可检索、可统计、可反思的个人认知资产。

## 二、当前阶段的核心定位

当前阶段明确采用“账本优先、复盘优先、自进化优先”的路线：

1. 先把交易计划、交易事实、情绪失误、经验沉淀记下来
2. 再让 OpenClaw / 大模型在会话里逐步补齐缺失信息
3. 再把这些历史轨迹转成优质路径、风险基因和行为提醒
4. 新闻 / 公告 / 晨报只作为辅助信息层，不抢主线

## 三、关键设计哲学：认知镜头，而不是上帝视角

这个框架不假装自己知道“真正的市场”是什么，也不试图用一个外部真相去否定用户当时的主观判断。

它要记录的是：

- 用户当时看向哪里：`user_focus`
- 用户当时看到了什么：`observed_signals`
- 用户如何解读：`thesis / logic_tags / environment_tags`
- 用户据此做了什么：计划、交易、仓位、等待或放弃

所以，这套框架记录的是“用户视角下的市场切片”，而不是“全市场全知全能监控”。

这也是为什么当前实现会把这些内容沉淀进 `decision_context_json`，并在 Vault 里单独写成“用户视角快照”。

对于量化 / 半量化场景，当前也允许把 `strategy_context` 单独挂进 `decision_context_json`，把：

- 策略条线
- 因子组
- 参数版本
- 启用原因
- 主观覆盖动作

作为另一条平行的“认知镜头”保存下来。

## 四、当前已经形成的最小闭环

围绕这个目的，当前已经形成的最小闭环是：

- 非标准化自然语言记账
- 多轮草稿轮询补齐
- 共享上下文提示与并行补问分组
- OpenClaw 会话级状态机
- 已落账记录的二次沉淀
- 用户视角快照持久化
- 自进化报告与提醒
- 风格画像卡片
- 行为体检报告
- URL-only 新闻轮询与晨报

这意味着系统已经能做到：

- 先快速落事实
- 再慢慢补认知切片和软结构标签
- 对同日环境 / 同票做T / 同策略条线，尽量复用共享上下文，减少重复轮询
- 再从历史里抽取优质路径和风险基因
- 再在下一次计划 / 交易时反过来提醒用户

## 五、为什么这里的结构化是“软结构”

这套框架里的结构化，目的不是为了自动交易。

它服务的是：

- 索引
- 检索
- 聚类
- 统计
- 自我反思

所以这里的标签、路径、基因，本质上都是“软结构”：

- 帮用户记住自己是怎么想的
- 帮用户检索过去类似场景
- 帮用户识别自己反复犯的错误
- 帮用户把“当时的自己”摆回“现在的自己”面前

但它们不应该直接变成自动下单规则。

## 六、新闻辅助层在这套框架里的位置

新闻 / 公告 / 晨报很重要，但它们的角色是：

- 给当天的计划与交易补环境
- 给晨报和展示提供稳定输入源
- 让云端 OpenClaw 即使没有浏览器自动化，也能只凭 URL 抓到可用事件

它们不是：

- 自动买卖指令
- 替用户决策的“客观答案”

## 七、下一阶段真正要推进的方向

在当前最小闭环之上，下一阶段真正要推进的是：

- 更强的大模型轮询代理
- 会话级共享上下文缓存与批量复用
- 更稳定的语义标签归一化
- 风格画像版本化
- 轨迹相似度与阶段性变化检测
- 量化策略轨迹的独立记账、自进化与主观轨迹对照
- 从多笔交易里自动蒸馏出原则卡片、禁做清单、白皮书草稿

换句话说，未来目标不是“记更多数据”，而是“让数据越来越像你的认知镜子”。

## 八、更长远的愿景

更长远地看，这个框架希望从“个人账本”走向“公共知识资产”：

- 每个人先拥有自己的 Agent 账本
- 在授权下，把经验蒸馏成匿名化经验卡片
- 再进一步形成面向大众的交易思想白皮书社区

这不是为了制造新的“股神神话”，而是为了沉淀：

- 有样本的经验
- 有边界的结论
- 有失败案例的纪律原则
- 可以被持续修正的公共认知资产

## 九、当前阶段的判断标准

这套框架做得好不好，不看它说了多少市场观点，而看它能否持续做到：

- 让记账更容易
- 让复盘更真实
- 让提醒更像“复用自己的历史”，而不是“听 AI 给建议”
- 让新闻和晨报更像环境补充，而不是主线替代品

这就是它当前最核心的目的，也是未来所有扩展都不应偏离的方向。
