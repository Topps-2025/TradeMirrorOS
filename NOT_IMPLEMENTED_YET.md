# 未实现 / 待实现能力清单

更新日期：2026-04-12

统一状态、已实现项与版本更新请优先查看：`FINANCE_JOURNAL_STATUS_AND_CHANGELOG.md`

## 一、真正的大模型多轮记账代理（部分未实现）

当前已实现的是：

- 启发式自由文本解析
- 缺失字段识别
- 自动补问建议
- 持久化草稿状态机
- 四轴轮询补齐元数据
- `shared_context_hints / parallel_question_groups` 这类“可复用 / 可并行补问”提示
- 面向 OpenClaw 的会话级入口：`session turn / state / reset`
- 已落账记录的二次沉淀入口：`plan enrich / trade enrich`

但还没有完整实现：

- 大模型驱动的语义问答状态机
- 真正理解“这句话是在回答上一问还是在修正之前记录”的槽位回填
- 根据个人习惯动态调整四轴补问顺序
- 会话内真正自动缓存“同日环境 / 同票主线 / 同策略条线”，并在后续多条记录里直接复用
- 自动把一组相似问题合并成一次批量提问，而不是目前仍主要依赖上层读取 `parallel_question_groups`
- 跨设备上下文记忆与中断恢复
- 把 `follow_up_questions` 直接接成连续对话闭环，而不是当前的“解析结果 + 人工再调用”
- 跨设备共享的 `session_key` / `session_memory` 合并与恢复策略

## 二、更强的语义标签抽取（未实现）

当前标签抽取主要基于规则和关键词。

还没有实现：

- 基于大模型的语义标签归一化
- 从长段复盘文本中抽取“核心交易思想”
- 从散乱、多跳、反复修正的对话里稳定抽取“最终版本”的标准标签
- 自动识别“这是纪律问题 / 认知问题 / 执行问题 / 情绪问题”
- 基于个人历史风格自动建议更贴切的标签
- 对旧记录二次补充时做更强的语义去重与纠偏

## 三、更深的风格建模与轨迹自进化（未实现）

当前已做的基础是：

- 从闭环交易中提取优质路径
- 提取可复用基因 / 风险基因
- 用 bandit 风格排序层给路径 / 基因加上 posterior / UCB / risk penalty
- 把结果转成提醒接口、复核清单与反思追问
- 输出基础风格画像卡片
- 支持把量化 / 半量化策略上下文（策略条线、因子、参数版本、启用原因）先落进 `decision_context_json.strategy_context`

但还没有做到：

- 风格画像的长期演化版本管理
- 不同周期、不同市场阶段下的风格切换检测
- 按“轨迹相似度”而不是简单标签重合来寻找最佳样本
- 按“策略条线 / 因子组 / 参数版本”拆开的独立量化轨迹统计
- 量化轨迹与主观交易轨迹的对照分析
- 策略启停原因、主观覆盖动作与参数漂移的阶段性报告
- 将盘口截图、分时图、订单明细纳入轨迹基因
- 真正的 contextual bandit 在线更新闭环
- 基于更细粒度轨迹的 offline RL / trajectory policy
- 把体检结果、卖出后回顾和阶段性补充自动收敛成“当前风格变化报告”

## 五、协作 / 社区层（未实现）

还没有实现：

- 多人共享账本社区
- Agent 之间共享匿名化经验卡片
- 按风格聚类的公共知识资产库
- 面向大众的交易思想白皮书社区
- 社区版本的规则投票、溯源、审校和演化机制

## 六、数据与执行层（未实现）

还没有实现：

- 券商 / 持仓 / 账户 API 自动导入
- 精准仓位管理与真实资金曲线
- 真实逐笔交割流水回放
- 分时级行为还原
- 图像 / 盘口截图自动归档
- 真正的自动任务编排和错误恢复中心
- 盘中级 `state-action-reward` 轨迹采集；没有这层，就不适合贸然把自进化升级成完整 RL
- 展示脚本虽然可生成“真实行情 + 模拟理由”的 demo 账本，但并不等同真实交易流水

补充说明：

- 标准化 CSV / JSON 交割单导入已经支持，但当前仍属于“文件级事实对齐”，不是券商账户级自动同步
- Git 层面的代码同步说明已补齐，但账本记录级 merge / conflict resolution 还没有真正实现

## 七、信息监测仍待增强的部分（未实现）

当前已经能做到 5 站点 URL-only 新闻抓取和晨报前轮询，但还没有实现：

- 更成熟的公告 / 新闻 / 宏观事件聚合策略
- 更稳的跨站去重、摘要归并与重要度排序
- 与记账会话完全打通的信息提醒编排
- 更细粒度的 source-level 质量评分与自动降权
- 站点失效后的自动切换 / 自动告警

## 八、为什么这些暂时没做

这轮优先级刻意收敛为：

1. 先把“记账”变得容易
2. 先把“复盘”变得真实
3. 先把“用户视角快照 -> 自进化 -> 体检”形成最小闭环
4. 再把新闻晨报与 OpenClaw 轮询入口打通

同时，这轮也刻意避免做“上帝视角”的全市场监控。

当前框架只记录：

- 用户当时看向了哪些标的 / 板块 / 线索
- 用户当时看到了什么信号
- 用户如何解读这些信号
- 用户据此做了什么行动

而把更重的大模型代理、在线强化学习、社区治理和真实资金流水接入留在后面。
